# Вариант 0

## Кейс: Автоматизация отбора сотрудников в корпоративную академию

В крупной компании запускается внутренняя корпоративная академия.
Руководство приняло решение ежегодно отбирать сотрудников для участия в программе обучения, направленной на развитие управленческих и профессиональных компетенций.

Ранее отбор проводился вручную: HR-специалисты анализировали возраст сотрудников, их рабочие показатели, количество выполненных задач и допущенных ошибок. Процесс занимал несколько дней и сопровождался риском ошибок при подсчётах.

В этом году принято решение автоматизировать процесс с использованием программного робота, разработанного в **Puzzle RPA**.

HR-отдел формирует список заявок сотрудников и передаёт его в виде JSON-файла.
Задача разработчика — спроектировать робота с использованием функций, который:

* корректно считает данные из файла;
* структурированно обрабатывает список сотрудников;
* разделяет логику на самостоятельные модули;
* формирует новые списки результатов;
* сохраняет итоговые данные в отдельные JSON-файлы.

---

# Требования к исходным данным

1. Исходные данные необходимо **сгенерировать с помощью ИИ DeepSeek**.
2. Сформировать JSON-файл, содержащий **ровно 6 записей**.
3. Формат — список словарей.
4. Структура каждой записи:

```json
{
  "name": "Иванов И.И.",
  "age": 25,
  "tasks_completed": 40,
  "errors": 3
}
```

5. Робот должен считать данные из созданного JSON-файла.

---

# 🔹 Задание 1 — Базовый уровень

## Этап 1: Проверка допуска к обучению

### Ситуация

На первом этапе руководство установило базовое требование:
к обучению допускаются только совершеннолетние сотрудники.

HR-отделу необходимо получить отдельный файл со статусом допуска по каждому сотруднику.

---

## Требуется:

1. Считать JSON-файл (6 сотрудников).
2. Организовать цикл обработки списка словарей.
3. Создать функцию с возвратом значения.

---

### Функция `CheckEmployeeAge`

**Тип:** Определение функции и возврат значения
**Параметр:**

* `employee_dict`

**Логика:**

* Получить значение `age` из словаря.
* Если возраст < 18 → вернуть `"Недопущен"`.
* Если возраст ≥ 18 → вернуть `"Допущен"`.

---

## Главный процесс должен:

1. Создать пустой список `admission_result_list`.
2. В цикле:

   * Вызвать `CheckEmployeeAge`;
   * Сформировать словарь:

```json
{
  "name": "Иванов И.И.",
  "admission_status": "Допущен"
}
```

* Добавить словарь в `admission_result_list`.

3. Сохранить `admission_result_list` в файл
   `admission_result.json`.
4. Вывести итоговый список на экран.

---

## Архитектурные требования

* Функция создаётся через блок «Определение функции и возврат значения».
* Функция выполняет только одну задачу — проверку возраста.
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит логики проверки.

---

# 🔹 Задание 2 — Повышенный уровень

## Этап 2: Формирование списка рекомендованных к обучению

### Ситуация

После проверки возраста руководство компании решило усложнить процесс отбора.

Теперь требуется учитывать рабочую эффективность сотрудников.
Было принято решение:

* оценивать соотношение выполненных задач к количеству ошибок;
* формировать рекомендацию на основании коэффициента эффективности;
* создать отдельный итоговый файл с результатами второго этапа отбора.

HR-отделу нужен структурированный список сотрудников со статусом рекомендации для передачи руководителям подразделений.

---

## Требуется:

1. Использовать исходный JSON-файл из 6 сотрудников.
2. Организовать цикл обработки списка.
3. Создать три функции с возвратом значения.

---

### 1️⃣ Функция `CalculatePerformance`

**Параметр:**

* `employee_dict`

**Логика:**

* Получить `tasks_completed` и `errors`.
* Если `tasks_completed` = 0 → вернуть 0.
* Рассчитать коэффициент:
  `(tasks_completed - errors) / tasks_completed`
* Вернуть число.

---

### 2️⃣ Функция `DetermineRecommendation`

**Параметр:**

* `performance_value`

**Логика:**

* Если коэффициент ≥ 0.85 → вернуть `"Рекомендован"`
* Иначе → вернуть `"Не рекомендован"`

---

### 3️⃣ Функция `BuildResultRecord`

**Параметры:**

* `employee_name`
* `recommendation_text`

**Логика:**

* Создать и вернуть словарь:

```json
{
  "name": "Иванов И.И.",
  "status": "Рекомендован"
}
```

---

## Главный процесс должен:

1. Создать пустой список `final_result_list`.
2. В цикле для каждого сотрудника:

   * Вызвать `CalculatePerformance`;
   * Передать результат в `DetermineRecommendation`;
   * Вызвать `BuildResultRecord`;
   * Добавить возвращённый словарь в `final_result_list`.
3. Сохранить `final_result_list` в файл
   `recommendation_result.json`.
4. Проверить, что итоговый файл содержит **6 записей**.
5. Вывести итоговый список на экран.

---

## Архитектурные требования

* Все функции создаются через блок «Определение функции и возврат значения».
* Каждая функция реализует принцип «одна функция — одна ответственность».
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит вычислений.
* Используются списки и словари.
* Все результаты обязательно сохраняются в файл.
