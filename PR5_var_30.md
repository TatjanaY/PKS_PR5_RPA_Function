# Вариант 30

## Кейс: Автоматизация формирования списка студентов, рекомендованных на стипендию

В крупном учебном заведении внедряется система автоматического отбора студентов для назначения повышенной академической стипендии.
Руководство приняло решение исключить ручную обработку данных об успеваемости и внеучебной активности, минимизировав риски субъективной оценки и ошибок при ранжировании кандидатов.

Ранее стипендиальная комиссия вручную анализировала ведомости с оценками, отчёты о научных достижениях и социальной активности студентов, формируя рейтинговые списки в таблицах. Процесс занимал значительное время перед каждым назначением стипендии и сопровождался риском несогласованности критериев отбора.

В этом году принято решение автоматизировать процесс с использованием программного робота, разработанного в **Puzzle RPA**.

Учебный отдел формирует список студентов с данными об успеваемости и достижениях и передаёт его в виде JSON-файла.
Задача разработчика — спроектировать робота с использованием функций, который:

* корректно считывает данные из файла;
* структурированно обрабатывает список студентов;
* разделяет логику на самостоятельные модули;
* формирует новые списки результатов;
* сохраняет итоговые данные в отдельные JSON-файлы.

---

# Требования к исходным данным

1. Исходные данные необходимо **сгенерировать с помощью ИИ DeepSeek**.
2. Сформировать JSON-файл, содержащий **ровно 6 записей**.
3. Формат — список словарей.
4. Структура каждой записи:

```json
{
  "student_name": "Васильев В.В.",
  "gpa": 4.8,
  "scientific_points": 15,
  "social_activity_hours": 40
}
```

5. Робот должен считать данные из созданного JSON-файла.

---

# 🔹 Задание 1 — Базовый уровень

## Этап 1: Проверка минимального порога успеваемости

### Ситуация

На первом этапе система должна проверить, достигает ли студент минимального порога среднего балла для рассмотрения на стипендию.
Согласно положению о стипендиальном обеспечении, минимальный порог GPA составляет 4.5.

Учебному отделу необходимо получить отдельный файл со статусом соответствия порогу успеваемости для каждого студента.

---

## Требуется:

1. Считать JSON-файл (6 студентов).
2. Организовать цикл обработки списка словарей.
3. Создать функцию с возвратом значения.

---

### Функция `CheckGpaEligibility`

**Тип:** Определение функции и возврат значения
**Параметр:**

* `student_dict`

**Логика:**

* Получить значение `gpa` из словаря.
* Если `gpa` < 4.5 → вернуть `"Не соответствует"`.
* Если `gpa` ≥ 4.5 → вернуть `"Соответствует"`.

---

## Главный процесс должен:

1. Создать пустой список `gpa_check_list`.
2. В цикле:

   * Вызвать `CheckGpaEligibility`;
   * Сформировать словарь:

```json
{
  "student_name": "Васильев В.В.",
  "gpa_status": "Соответствует"
}
```

* Добавить словарь в `gpa_check_list`.

3. Сохранить `gpa_check_list` в файл
   `gpa_check.json`.
4. Вывести итоговый список на экран.

---

## Архитектурные требования

* Функция создаётся через блок «Определение функции и возврат значения».
* Функция выполняет только одну задачу — проверку порога успеваемости.
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит логики проверки.

---

# 🔹 Задание 2 — Повышенный уровень

## Этап 2: Расчёт стипендиального рейтинга и формирование итогового списка

### Ситуация

После проверки успеваемости руководство решило автоматизировать комплексную оценку кандидатов на стипендию.

Теперь требуется не только подтвердить соответствие порогу GPA, но и рассчитать интегральный стипендиальный балл на основе трёх факторов (успеваемость, научная активность, социальная вовлечённость), а также присвоить студенту уровень стипендии.

Стипендиальной комиссии нужен структурированный список студентов с расчётными данными для принятия решения о назначении выплат.

---

## Требуется:

1. Использовать исходный JSON-файл из 6 студентов.
2. Организовать цикл обработки списка.
3. Создать три функции с возвратом значения.

---

### 1️⃣ Функция `CalculateScholarshipScore`

**Параметр:**

* `student_dict`

**Логика:**

* Получить `gpa`, `scientific_points`, `social_activity_hours`.
* Рассчитать стипендиальный балл по формуле:
  `(gpa * 10) + (scientific_points * 0.5) + (social_activity_hours * 0.1)`
* Вернуть число (интегральный рейтинг).

---

### 2️⃣ Функция `DetermineScholarshipLevel`

**Параметр:**

* `scholarship_score`

**Логика:**

* Если `scholarship_score` ≥ 55 → вернуть `"Повышенная стипендия"`
* Если `scholarship_score` ≥ 50 → вернуть `"Стандартная стипендия"`
* Иначе → вернуть `"Базовая стипендия"`

---

### 3️⃣ Функция `BuildScholarshipRecord`

**Параметры:**

* `student_name`
* `scholarship_level`
* `score_value`

**Логика:**

* Создать и вернуть словарь:

```json
{
  "student_name": "Васильев В.В.",
  "scholarship_level": "Повышенная стипендия",
  "scholarship_score": 58.5
}
```

---

## Главный процесс должен:

1. Создать пустой список `final_scholarship_list`.
2. В цикле для каждого студента:

   * Вызвать `CheckGpaEligibility` (проверить статус GPA);
   * Вызвать `CalculateScholarshipScore`;
   * Передать результат в `DetermineScholarshipLevel`;
   * Вызвать `BuildScholarshipRecord`;
   * Добавить возвращённый словарь в `final_scholarship_list`.
3. Сохранить `final_scholarship_list` в файл
   `scholarship_result.json`.
4. Проверить, что итоговый файл содержит **6 записей**.
5. Вывести итоговый список на экран.

---

## Архитектурные требования

* Все функции создаются через блок «Определение функции и возврат значения».
* Каждая функция реализует принцип «одна функция — одна ответственность».
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит вычислений.
* Используются списки и словари.
* Все результаты обязательно сохраняются в файл.
