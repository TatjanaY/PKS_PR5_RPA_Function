# Вариант 28

## Кейс: Автоматизация распределения студентов по учебным группам

В крупном учебном заведении внедряется система автоматического распределения студентов по учебным группам на основе их академических показателей и предпочтений.
Руководство приняло решение исключить ручное формирование групп и минимизировать риски несбалансированного распределения студентов по уровню подготовки.

Ранее методисты вручную анализировали средний балл, направление подготовки и пожелания студентов по расписанию, формируя группы в таблицах. Процесс занимал значительное время перед началом семестра и сопровождался риском субъективной оценки и дисбаланса уровней в группах.

В этом году принято решение автоматизировать процесс с использованием программного робота, разработанного в **Puzzle RPA**.

Учебный отдел формирует список студентов с данными об успеваемости и предпочтениях и передаёт его в виде JSON-файла.
Задача разработчика — спроектировать робота с использованием функций, который:

* корректно считывает данные из файла;
* структурированно обрабатывает список студентов;
* разделяет логику на самостоятельные модули;
* формирует новые списки результатов;
* сохраняет итоговые данные в отдельные JSON-файлы.

---

# Требования к исходным данным

1. Исходные данные необходимо **сгенерировать с помощью ИИ DeepSeek**.
2. Сформировать JSON-файл, содержащий **ровно 6 записей**.
3. Формат — список словарей.
4. Структура каждой записи:

```json
{
  "student_name": "Соколов С.С.",
  "gpa": 4.5,
  "preferred_schedule": "утро",
  "specialization": "программирование"
}
```

5. Робот должен считать данные из созданного JSON-файла.

---

# 🔹 Задание 1 — Базовый уровень

## Этап 1: Проверка соответствия специализации

### Ситуация

На первом этапе система должна проверить, соответствует ли заявленная специализация студента доступным учебным направлениям.
Доступные направления: "программирование", "дизайн", "аналитика".

Учебному отделу необходимо получить отдельный файл со статусом соответствия специализации для каждого студента.

---

## Требуется:

1. Считать JSON-файл (6 студентов).
2. Организовать цикл обработки списка словарей.
3. Создать функцию с возвратом значения.

---

### Функция `CheckSpecializationValidity`

**Тип:** Определение функции и возврат значения
**Параметр:**

* `student_dict`

**Логика:**

* Получить значение `specialization` из словаря.
* Если специализация не входит в список ["программирование", "дизайн", "аналитика"] → вернуть `"Неверная специализация"`.
* Иначе → вернуть `"Специализация подтверждена"`.

---

## Главный процесс должен:

1. Создать пустой список `specialization_check_list`.
2. В цикле:

   * Вызвать `CheckSpecializationValidity`;
   * Сформировать словарь:

```json
{
  "student_name": "Соколов С.С.",
  "specialization_status": "Специализация подтверждена"
}
```

* Добавить словарь в `specialization_check_list`.

3. Сохранить `specialization_check_list` в файл
   `specialization_check.json`.
4. Вывести итоговый список на экран.

---

## Архитектурные требования

* Функция создаётся через блок «Определение функции и возврат значения».
* Функция выполняет только одну задачу — проверку валидности специализации.
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит логики проверки.

---

# 🔹 Задание 2 — Повышенный уровень

## Этап 2: Расчёт приоритета зачисления и формирование группы

### Ситуация

После проверки специализации руководство решило автоматизировать распределение студентов по группам с учётом академических показателей и предпочтений.

Теперь требуется не только подтвердить специализацию, но и рассчитать приоритетный балл для зачисления в группу, а также присвоить студенту номер учебной группы на основе расписания и уровня подготовки.

Деканату нужен структурированный список студентов с назначенными группами для формирования учебных планов.

---

## Требуется:

1. Использовать исходный JSON-файл из 6 студентов.
2. Организовать цикл обработки списка.
3. Создать три функции с возвратом значения.

---

### 1️⃣ Функция `CalculateGroupPriority`

**Параметр:**

* `student_dict`

**Логика:**

* Получить `gpa` и `preferred_schedule`.
* Базовый приоритет: `gpa * 2`.
* Если `preferred_schedule` = "утро" → добавить 1 балл (дефицитное время).
* Вернуть число (приоритетный балл).

---

### 2️⃣ Функция `DetermineGroupNumber`

**Параметры:**

* `specialization` (строка)
* `priority_score` (число)

**Логика:**

* Определить базовый номер группы по специализации:
  - "программирование" → 100
  - "дизайн" → 200
  - "аналитика" → 300
* Если `priority_score` ≥ 9 → добавить суффикс "А" (продвинутая группа).
* Иначе → добавить суффикс "Б" (базовая группа).
* Вернуть строку с номером группы (например, "100А").

---

### 3️⃣ Функция `BuildGroupRecord`

**Параметры:**

* `student_name`
* `group_number`
* `priority_value`

**Логика:**

* Создать и вернуть словарь:

```json
{
  "student_name": "Соколов С.С.",
  "assigned_group": "100А",
  "priority_score": 10.0
}
```

---

## Главный процесс должен:

1. Создать пустой список `final_group_list`.
2. В цикле для каждого студента:

   * Вызвать `CheckSpecializationValidity` (проверить статус);
   * Вызвать `CalculateGroupPriority`;
   * Вызвать `DetermineGroupNumber` с двумя параметрами;
   * Вызвать `BuildGroupRecord`;
   * Добавить возвращённый словарь в `final_group_list`.
3. Сохранить `final_group_list` в файл
   `group_assignment_result.json`.
4. Проверить, что итоговый файл содержит **6 записей**.
5. Вывести итоговый список на экран.

---

## Архитектурные требования

* Все функции создаются через блок «Определение функции и возврат значения».
* Каждая функция реализует принцип «одна функция — одна ответственность».
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит вычислений.
* Используются списки и словари.
* Все результаты обязательно сохраняются в файл.
