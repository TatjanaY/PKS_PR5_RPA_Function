# Вариант 25

## Кейс: Автоматизация формирования рейтинга студентов

В крупном учебном заведении внедряется система автоматического расчёта рейтингов студентов по итогам семестра.
Руководство приняло решение исключить ручную обработку ведомостей и минимизировать риски ошибок при подсчёте средних баллов и определении мест в рейтинге.

Ранее сотрудники деканата вручную собирали данные об оценках, посещаемости и активности студентов, рассчитывали итоговые баллы и формировали рейтинговые списки. Процесс занимал значительное время в конце семестра и сопровождался риском арифметических ошибок.

В этом году принято решение автоматизировать процесс с использованием программного робота, разработанного в **Puzzle RPA**.

Учебный отдел формирует список студентов с данными об успеваемости и передаёт его в виде JSON-файла.
Задача разработчика — спроектировать робота с использованием функций, который:

* корректно считывает данные из файла;
* структурированно обрабатывает список студентов;
* разделяет логику на самостоятельные модули;
* формирует новые списки результатов;
* сохраняет итоговые данные в отдельные JSON-файлы.

---

# Требования к исходным данным

1. Исходные данные необходимо **сгенерировать с помощью ИИ DeepSeek**.
2. Сформировать JSON-файл, содержащий **ровно 6 записей**.
3. Формат — список словарей.
4. Структура каждой записи:

```json
{
  "student_name": "Алексеев А.А.",
  "group": "ИВТ-202",
  "average_grade": 4.5,
  "attendance_percent": 85,
  "activity_points": 10
}
```

5. Робот должен считать данные из созданного JSON-файла.

---

# 🔹 Задание 1 — Базовый уровень

## Этап 1: Проверка допуска к рейтингу

### Ситуация

На первом этапе система должна проверить, имеет ли студент допуск к участию в рейтинге.
Студенты с посещаемостью ниже 70% не допускаются к итоговому рейтингу.

Учебному отделу необходимо получить отдельный файл со статусом допуска по каждому студенту.

---

## Требуется:

1. Считать JSON-файл (6 студентов).
2. Организовать цикл обработки списка словарей.
3. Создать функцию с возвратом значения.

---

### Функция `CheckRatingEligibility`

**Тип:** Определение функции и возврат значения
**Параметр:**

* `student_dict`

**Логика:**

* Получить значение `attendance_percent` из словаря.
* Если `attendance_percent` < 70 → вернуть `"Не допущен"`.
* Если `attendance_percent` ≥ 70 → вернуть `"Допущен"`.

---

## Главный процесс должен:

1. Создать пустой список `eligibility_check_list`.
2. В цикле:

   * Вызвать `CheckRatingEligibility`;
   * Сформировать словарь:

```json
{
  "student_name": "Алексеев А.А.",
  "eligibility_status": "Допущен"
}
```

* Добавить словарь в `eligibility_check_list`.

3. Сохранить `eligibility_check_list` в файл
   `eligibility_check.json`.
4. Вывести итоговый список на экран.

---

## Архитектурные требования

* Функция создаётся через блок «Определение функции и возврат значения».
* Функция выполняет только одну задачу — проверку допуска к рейтингу.
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит логики проверки.

---

# 🔹 Задание 2 — Повышенный уровень

## Этап 2: Расчёт итогового рейтингового балла и присвоение места

### Ситуация

После проверки допуска руководство решило усложнить процесс формирования рейтинга.

Теперь требуется рассчитать итоговый рейтинговый балл на основе среднего балла, посещаемости и активности, а также присвоить студенту категорию успеваемости.

Деканату нужен структурированный список студентов с рейтинговыми баллами для определения стипендий и поощрений.

---

## Требуется:

1. Использовать исходный JSON-файл из 6 студентов.
2. Организовать цикл обработки списка.
3. Создать три функции с возвратом значения.

---

### 1️⃣ Функция `CalculateRatingScore`

**Параметр:**

* `student_dict`

**Логика:**

* Получить `average_grade`, `attendance_percent`, `activity_points`.
* Рассчитать балл:
  `(average_grade * 20) + (attendance_percent / 10) + activity_points`
* Вернуть число.

---

### 2️⃣ Функция `DeterminePerformanceCategory`

**Параметр:**

* `score_value`

**Логика:**

* Если `score_value` ≥ 120 → вернуть `"Отличник"`
* Если `score_value` ≥ 100 → вернуть `"Хорошист"`
* Иначе → вернуть `"Требуется внимание"`

---

### 3️⃣ Функция `BuildRatingRecord`

**Параметры:**

* `student_name`
* `category_text`
* `rating_score`

**Логика:**

* Создать и вернуть словарь:

```json
{
  "student_name": "Алексеев А.А.",
  "performance_category": "Отличник",
  "rating_score": 125
}
```

---

## Главный процесс должен:

1. Создать пустой список `final_rating_list`.
2. В цикле для каждого студента:

   * Вызвать `CalculateRatingScore`;
   * Передать результат в `DeterminePerformanceCategory`;
   * Вызвать `BuildRatingRecord`;
   * Добавить возвращённый словарь в `final_rating_list`.
3. Сохранить `final_rating_list` в файл
   `rating_result.json`.
4. Проверить, что итоговый файл содержит **6 записей**.
5. Вывести итоговый список на экран.

---

## Архитектурные требования

* Все функции создаются через блок «Определение функции и возврат значения».
* Каждая функция реализует принцип «одна функция — одна ответственность».
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит вычислений.
* Используются списки и словари.
* Все результаты обязательно сохраняются в файл.
