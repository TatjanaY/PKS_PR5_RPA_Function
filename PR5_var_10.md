# Вариант 10

## Кейс: Автоматизация расчёта скидок для постоянных клиентов

В крупной розничной компании внедряется система автоматического расчёта персональных скидок для клиентов на основе их лояльности и истории покупок.
Руководство приняло решение исключить ручные вычисления при формировании скидочных программ и минимизировать риски ошибок при начислении бонусов.

Ранее менеджеры вручную анализировали суммы покупок, количество заказов и стаж клиента в системе, рассчитывали процент скидки в таблицах. Процесс занимал значительное время и сопровождался риском арифметических ошибок и субъективной оценки.

В этом году принято решение автоматизировать процесс с использованием программного робота, разработанного в **Puzzle RPA**.

Отдел лояльности формирует список клиентов с данными о покупках и передаёт его в виде JSON-файла.
Задача разработчика — спроектировать робота с использованием функций, который:

* корректно считывает данные из файла;
* структурированно обрабатывает список клиентов;
* разделяет логику на самостоятельные модули;
* формирует новые списки результатов;
* сохраняет итоговые данные в отдельные JSON-файлы.

---

# Требования к исходным данным

1. Исходные данные необходимо **сгенерировать с помощью ИИ DeepSeek**.
2. Сформировать JSON-файл, содержащий **ровно 6 записей**.
3. Формат — список словарей.
4. Структура каждой записи:

```json
{
  "client_name": "Смирнова А.А.",
  "total_purchases": 150000,
  "order_count": 12,
  "loyalty_years": 3
}
```

5. Робот должен считать данные из созданного JSON-файла.

---

# 🔹 Задание 1 — Базовый уровень

## Этап 1: Проверка права на скидку

### Ситуация

На первом этапе система должна проверить, имеет ли клиент право на получение скидки.
Право на скидку получают клиенты, совершившие не менее 10 заказов за весь период сотрудничества.

Отделу лояльности необходимо получить отдельный файл со статусом права на скидку по каждому клиенту.

---

## Требуется:

1. Считать JSON-файл (6 клиентов).
2. Организовать цикл обработки списка словарей.
3. Создать функцию с возвратом значения.

---

### Функция `CheckDiscountEligibility`

**Тип:** Определение функции и возврат значения
**Параметр:**

* `client_dict`

**Логика:**

* Получить значение `order_count` из словаря.
* Если `order_count` < 10 → вернуть `"Не имеет права"`.
* Если `order_count` ≥ 10 → вернуть `"Имеет право"`.

---

## Главный процесс должен:

1. Создать пустой список `eligibility_result_list`.
2. В цикле:

   * Вызвать `CheckDiscountEligibility`;
   * Сформировать словарь:

```json
{
  "client_name": "Смирнова А.А.",
  "discount_eligibility": "Имеет право"
}
```

* Добавить словарь в `eligibility_result_list`.

3. Сохранить `eligibility_result_list` в файл
   `eligibility_result.json`.
4. Вывести итоговый список на экран.

---

## Архитектурные требования

* Функция создаётся через блок «Определение функции и возврат значения».
* Функция выполняет только одну задачу — проверку права на скидку.
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит логики проверки.

---

# 🔹 Задание 2 — Повышенный уровень

## Этап 2: Расчёт размера скидки и формирование итогового предложения

### Ситуация

После проверки права на скидку руководство решило автоматизировать расчёт персонального процента скидки.

Теперь требуется не только подтвердить право, но и рассчитать конкретный размер скидки на основе суммы покупок и стажа лояльности, а также сформировать итоговое коммерческое предложение.

Отделу маркетинга нужен структурированный список клиентов с расчётными данными для рассылки персональных предложений.

---

## Требуется:

1. Использовать исходный JSON-файл из 6 клиентов.
2. Организовать цикл обработки списка.
3. Создать три функции с возвратом значения.

---

### 1️⃣ Функция `CalculateDiscountPercent`

**Параметр:**

* `client_dict`

**Логика:**

* Получить `total_purchases` и `loyalty_years`.
* Базовый процент: 5%.
* Добавить 1% за каждые 50000 руб. покупок сверх 100000.
* Добавить 1% за каждый год лояльности сверх 2 лет.
* Ограничить максимальный процент значением 25%.
* Вернуть число (процент скидки).

---

### 2️⃣ Функция `DetermineDiscountTier`

**Параметр:**

* `discount_percent`

**Логика:**

* Если `discount_percent` ≥ 20 → вернуть `"Премиум-уровень"`
* Если `discount_percent` ≥ 10 → вернуть `"Стандартный уровень"`
* Иначе → вернуть `"Базовый уровень"`

---

### 3️⃣ Функция `BuildDiscountOffer`

**Параметры:**

* `client_name`
* `tier_label`
* `discount_value`

**Логика:**

* Создать и вернуть словарь:

```json
{
  "client_name": "Смирнова А.А.",
  "discount_tier": "Стандартный уровень",
  "discount_percent": 12
}
```

---

## Главный процесс должен:

1. Создать пустой список `final_discount_list`.
2. В цикле для каждого клиента:

   * Вызвать `CalculateDiscountPercent`;
   * Передать результат в `DetermineDiscountTier`;
   * Вызвать `BuildDiscountOffer`;
   * Добавить возвращённый словарь в `final_discount_list`.
3. Сохранить `final_discount_list` в файл
   `discount_result.json`.
4. Проверить, что итоговый файл содержит **6 записей**.
5. Вывести итоговый список на экран.

---

## Архитектурные требования

* Все функции создаются через блок «Определение функции и возврат значения».
* Каждая функция реализует принцип «одна функция — одна ответственность».
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит вычислений.
* Используются списки и словари.
* Все результаты обязательно сохраняются в файл.
