# Вариант 3

## Кейс: Автоматизация расчёта премий по KPI

В крупной компании внедряется система автоматического расчёта ежемесячных премий на основе ключевых показателей эффективности (KPI).
Руководство приняло решение исключить ручные вычисления из процесса начисления вознаграждений.

Ранее бухгалтеры вручную сверяли планы и факты, рассчитывали проценты выполнения и определяли суммы премий в Excel. Процесс занимал несколько дней в конце месяца и сопровождался риском арифметических ошибок.

В этом году принято решение автоматизировать процесс с использованием программного робота, разработанного в **Puzzle RPA**.

HR-отдел и отдел мотивации формируют список сотрудников с показателями и передают его в виде JSON-файла.
Задача разработчика — спроектировать робота с использованием функций, который:

* корректно считывает данные из файла;
* структурированно обрабатывает список сотрудников;
* разделяет логику на самостоятельные модули;
* формирует новые списки результатов;
* сохраняет итоговые данные в отдельные JSON-файлы.

---

# Требования к исходным данным

1. Исходные данные необходимо **сгенерировать с помощью ИИ DeepSeek**.
2. Сформировать JSON-файл, содержащий **ровно 6 записей**.
3. Формат — список словарей.
4. Структура каждой записи:

```json
{
  "name": "Кузнецов К.К.",
  "kpi_score": 85,
  "base_bonus": 50000
}
```

5. Робот должен считать данные из созданного JSON-файла.

---

# 🔹 Задание 1 — Базовый уровень

## Этап 1: Проверка права на премию

### Ситуация

На первом этапе система должна проверить, достиг ли сотрудник минимального порога выполнения KPI для получения любой премии.
Порог установлен на уровне 80%.

HR-отделу необходимо получить отдельный файл со статусом права на премию по каждому сотруднику.

---

## Требуется:

1. Считать JSON-файл (6 сотрудников).
2. Организовать цикл обработки списка словарей.
3. Создать функцию с возвратом значения.

---

### Функция `CheckKpiEligibility`

**Тип:** Определение функции и возврат значения
**Параметр:**

* `employee_dict`

**Логика:**

* Получить значение `kpi_score` из словаря.
* Если `kpi_score` < 80 → вернуть `"Не имеет права"`.
* Если `kpi_score` ≥ 80 → вернуть `"Имеет право"`.

---

## Главный процесс должен:

1. Создать пустой список `eligibility_result_list`.
2. В цикле:

   * Вызвать `CheckKpiEligibility`;
   * Сформировать словарь:

```json
{
  "name": "Кузнецов К.К.",
  "bonus_eligibility": "Имеет право"
}
```

* Добавить словарь в `eligibility_result_list`.

3. Сохранить `eligibility_result_list` в файл
   `eligibility_result.json`.
4. Вывести итоговый список на экран.

---

## Архитектурные требования

* Функция создаётся через блок «Определение функции и возврат значения».
* Функция выполняет только одну задачу — проверку порога KPI.
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит логики проверки.

---

# 🔹 Задание 2 — Повышенный уровень

## Этап 2: Расчёт итоговой суммы премии

### Ситуация

После проверки права на премию руководство решило автоматизировать расчёт итоговых выплат.

Теперь требуется не только подтвердить право, но и рассчитать конкретную сумму премии на основе базового бонуса и процента выполнения, а также присвоить уровень премии.

Отделу мотивации нужен структурированный список сотрудников с расчётными данными для передачи в бухгалтерию.

---

## Требуется:

1. Использовать исходный JSON-файл из 6 сотрудников.
2. Организовать цикл обработки списка.
3. Создать три функции с возвратом значения.

---

### 1️⃣ Функция `CalculateBonusAmount`

**Параметр:**

* `employee_dict`

**Логика:**

* Получить `base_bonus` и `kpi_score`.
* Рассчитать сумму:
  `base_bonus * (kpi_score / 100)`
* Вернуть число.

---

### 2️⃣ Функция `DetermineBonusLevel`

**Параметр:**

* `kpi_score`

**Логика:**

* Если `kpi_score` ≥ 100 → вернуть `"Максимальный уровень"`
* Если `kpi_score` ≥ 80 → вернуть `"Стандартный уровень"`
* Иначе → вернуть `"Минимальный уровень"`

---

### 3️⃣ Функция `BuildBonusRecord`

**Параметры:**

* `employee_name`
* `level_text`
* `amount_value`

**Логика:**

* Создать и вернуть словарь:

```json
{
  "name": "Кузнецов К.К.",
  "bonus_level": "Стандартный уровень",
  "final_amount": 42500
}
```

---

## Главный процесс должен:

1. Создать пустой список `final_bonus_list`.
2. В цикле для каждого сотрудника:

   * Вызвать `CalculateBonusAmount`;
   * Вызвать `DetermineBonusLevel` (передать `kpi_score`);
   * Вызвать `BuildBonusRecord`;
   * Добавить возвращённый словарь в `final_bonus_list`.
3. Сохранить `final_bonus_list` в файл
   `bonus_result.json`.
4. Проверить, что итоговый файл содержит **6 записей**.
5. Вывести итоговый список на экран.

---

## Архитектурные требования

* Все функции создаются через блок «Определение функции и возврат значения».
* Каждая функция реализует принцип «одна функция — одна ответственность».
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит вычислений.
* Используются списки и словари.
* Все результаты обязательно сохраняются в файл.
