# Вариант 29

## Кейс: Автоматизация расчёта среднего балла по дисциплине

В крупном учебном заведении внедряется система автоматического расчёта средних баллов студентов по отдельным дисциплинам для формирования академических справок и мониторинга успеваемости.
Руководство приняло решение исключить ручную обработку ведомостей и минимизировать риски ошибок при подсчёте итоговых оценок.

Ранее преподаватели вручную сводили данные о текущих оценках, экзаменационных результатах и дополнительных баллах за активность, рассчитывали средние значения и вносили их в журналы. Процесс занимал значительное время в конце семестра и сопровождался риском арифметических ошибок.

В этом году принято решение автоматизировать процесс с использованием программного робота, разработанного в **Puzzle RPA**.

Учебный отдел формирует список студентов с оценками по дисциплине и передаёт его в виде JSON-файла.
Задача разработчика — спроектировать робота с использованием функций, который:

* корректно считывает данные из файла;
* структурированно обрабатывает список студентов;
* разделяет логику на самостоятельные модули;
* формирует новые списки результатов;
* сохраняет итоговые данные в отдельные JSON-файлы.

---

# Требования к исходным данным

1. Исходные данные необходимо **сгенерировать с помощью ИИ DeepSeek**.
2. Сформировать JSON-файл, содержащий **ровно 6 записей**.
3. Формат — список словарей.
4. Структура каждой записи:

```json
{
  "student_name": "Козлов К.К.",
  "group": "ПМ-203",
  "current_marks": [4, 5, 4, 3, 5],
  "exam_mark": 5,
  "activity_bonus": 2
}
```

5. Робот должен считать данные из созданного JSON-файла.

---

# 🔹 Задание 1 — Базовый уровень

## Этап 1: Проверка наличия оценок для расчёта

### Ситуация

На первом этапе система должна проверить, есть ли у студента достаточное количество оценок для расчёта среднего балла.
Студенты без текущих оценок или экзаменационной отметки не могут быть включены в итоговую ведомость.

Учебному отделу необходимо получить отдельный файл со статусом готовности данных по каждому студенту.

---

## Требуется:

1. Считать JSON-файл (6 студентов).
2. Организовать цикл обработки списка словарей.
3. Создать функцию с возвратом значения.

---

### Функция `CheckDataCompleteness`

**Тип:** Определение функции и возврат значения
**Параметр:**

* `student_dict`

**Логика:**

* Получить значения `current_marks` (список) и `exam_mark` из словаря.
* Если список `current_marks` пуст ИЛИ `exam_mark` отсутствует (None/0) → вернуть `"Данные неполные"`.
* Если есть оценки → вернуть `"Данные полные"`.

---

## Главный процесс должен:

1. Создать пустой список `data_check_list`.
2. В цикле:

   * Вызвать `CheckDataCompleteness`;
   * Сформировать словарь:

```json
{
  "student_name": "Козлов К.К.",
  "data_status": "Данные полные"
}
```

* Добавить словарь в `data_check_list`.

3. Сохранить `data_check_list` в файл
   `data_check.json`.
4. Вывести итоговый список на экран.

---

## Архитектурные требования

* Функция создаётся через блок «Определение функции и возврат значения».
* Функция выполняет только одну задачу — проверку полноты данных.
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит логики проверки.

---

# 🔹 Задание 2 — Повышенный уровень

## Этап 2: Расчёт среднего балла и присвоение итоговой оценки

### Ситуация

После проверки данных руководство решило автоматизировать финальный расчёт по дисциплине.

Теперь требуется рассчитать взвешенный средний балл с учётом текущих оценок, экзамена и бонусов за активность, а также присвоить итоговую оценку по пятибалльной шкале.

Деканату нужен структурированный список студентов с итоговыми баллами для формирования академических справок.

---

## Требуется:

1. Использовать исходный JSON-файл из 6 студентов.
2. Организовать цикл обработки списка.
3. Создать три функции с возвратом значения.

---

### 1️⃣ Функция `CalculateAverageScore`

**Параметр:**

* `student_dict`

**Логика:**

* Получить `current_marks`, `exam_mark`, `activity_bonus`.
* Если список оценок пуст → вернуть 0.
* Рассчитать среднее текущих: `sum(current_marks) / len(current_marks)`.
* Рассчитать итоговый балл:
  `(среднее_текущих * 0.5) + (exam_mark * 0.4) + (activity_bonus * 0.1)`
* Вернуть число, округлённое до 2 знаков.

---

### 2️⃣ Функция `DetermineFinalGrade`

**Параметр:**

* `average_score`

**Логика:**

* Если `average_score` ≥ 4.5 → вернуть `"Отлично"`
* Если `average_score` ≥ 3.5 → вернуть `"Хорошо"`
* Если `average_score` ≥ 2.5 → вернуть `"Удовлетворительно"`
* Иначе → вернуть `"Неудовлетворительно"`

---

### 3️⃣ Функция `BuildGradeRecord`

**Параметры:**

* `student_name`
* `final_grade`
* `average_score`

**Логика:**

* Создать и вернуть словарь:

```json
{
  "student_name": "Козлов К.К.",
  "final_grade": "Отлично",
  "average_score": 4.65
}
```

---

## Главный процесс должен:

1. Создать пустой список `final_grades_list`.
2. В цикле для каждого студента:

   * Вызвать `CalculateAverageScore`;
   * Передать результат в `DetermineFinalGrade`;
   * Вызвать `BuildGradeRecord`;
   * Добавить возвращённый словарь в `final_grades_list`.
3. Сохранить `final_grades_list` в файл
   `grades_result.json`.
4. Проверить, что итоговый файл содержит **6 записей**.
5. Вывести итоговый список на экран.

---

## Архитектурные требования

* Все функции создаются через блок «Определение функции и возврат значения».
* Каждая функция реализует принцип «одна функция — одна ответственность».
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит вычислений.
* Используются списки и словари.
* Все результаты обязательно сохраняются в файл.
