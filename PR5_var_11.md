# Вариант 11

## Кейс: Автоматизация формирования отчёта по командировочным расходам

В крупной компании внедряется система автоматической обработки отчётов по командировочным расходам сотрудников.
Руководство приняло решение исключить ручную сверку авансовых отчётов и минимизировать риски ошибок при расчёте итоговых сумм к выплате или возврату.

Ранее бухгалтеры вручную проверяли чеки, сверяли дни командировки с суточными нормами и рассчитывали разницу между выданным авансом и фактическими расходами. Процесс занимал значительное время и сопровождался риском арифметических ошибок.

В этом году принято решение автоматизировать процесс с использованием программного робота, разработанного в **Puzzle RPA**.

Бухгалтерия формирует список отчётов сотрудников и передаёт его в виде JSON-файла.
Задача разработчика — спроектировать робота с использованием функций, который:

* корректно считывает данные из файла;
* структурированно обрабатывает список отчётов;
* разделяет логику на самостоятельные модули;
* формирует новые списки результатов;
* сохраняет итоговые данные в отдельные JSON-файлы.

---

# Требования к исходным данным

1. Исходные данные необходимо **сгенерировать с помощью ИИ DeepSeek**.
2. Сформировать JSON-файл, содержащий **ровно 6 записей**.
3. Формат — список словарей.
4. Структура каждой записи:

```json
{
  "employee_name": "Орлов О.О.",
  "trip_days": 5,
  "daily_allowance": 1000.00,
  "receipts_total": 15000.00,
  "advance_given": 20000.00
}
```

5. Робот должен считать данные из созданного JSON-файла.

---

# 🔹 Задание 1 — Базовый уровень

## Этап 1: Проверка наличия подтверждающих документов

### Ситуация

На первом этапе система должна проверить, приложил ли сотрудник подтверждающие документы (чеки) к отчёту.
Отчёты без чеков не могут быть приняты к обработке и требуют дополнения.

Бухгалтерии необходимо получить отдельный файл со статусом комплектности документов по каждому отчёту.

---

## Требуется:

1. Считать JSON-файл (6 отчётов).
2. Организовать цикл обработки списка словарей.
3. Создать функцию с возвратом значения.

---

### Функция `CheckDocumentsProvided`

**Тип:** Определение функции и возврат значения
**Параметр:**

* `report_dict`

**Логика:**

* Получить значение `receipts_total` из словаря.
* Если `receipts_total` = 0 → вернуть `"Документы отсутствуют"`.
* Если `receipts_total` > 0 → вернуть `"Документы предоставлены"`.

---

## Главный процесс должен:

1. Создать пустой список `documents_check_list`.
2. В цикле:

   * Вызвать `CheckDocumentsProvided`;
   * Сформировать словарь:

```json
{
  "employee_name": "Орлов О.О.",
  "documents_status": "Документы предоставлены"
}
```

* Добавить словарь в `documents_check_list`.

3. Сохранить `documents_check_list` в файл
   `documents_check.json`.
4. Вывести итоговый список на экран.

---

## Архитектурные требования

* Функция создаётся через блок «Определение функции и возврат значения».
* Функция выполняет только одну задачу — проверку наличия чеков.
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит логики проверки.

---

# 🔹 Задание 2 — Повышенный уровень

## Этап 2: Расчёт итоговой суммы и статуса расчётов

### Ситуация

После проверки документов руководство решило автоматизировать финальный расчёт по командировке.

Теперь требуется рассчитать общую сумму расходов (суточные + чеки), сравнить её с выданным авансом и определить статус расчётов (долг сотрудника, долг компании или погашено).

Бухгалтерии нужен структурированный список отчётов с итоговыми суммами для проведения платежей или требования возврата.

---

## Требуется:

1. Использовать исходный JSON-файл из 6 отчётов.
2. Организовать цикл обработки списка.
3. Создать три функции с возвратом значения.

---

### 1️⃣ Функция `CalculateTotalExpense`

**Параметр:**

* `report_dict`

**Логика:**

* Получить `trip_days`, `daily_allowance` и `receipts_total`.
* Рассчитать сумму суточных: `trip_days * daily_allowance`.
* Рассчитать общую сумму: `суточные + receipts_total`.
* Вернуть число.

---

### 2️⃣ Функция `DetermineSettlementStatus`

**Параметры:**

* `total_expense`
* `advance_given`

**Логика:**

* Если `total_expense` = `advance_given` → вернуть `"Расчёты погашены"`.
* Если `total_expense` > `advance_given` → вернуть `"Доплата сотруднику"`.
* Если `total_expense` < `advance_given` → вернуть `"Возврат в кассу"`.

---

### 3️⃣ Функция `BuildExpenseRecord`

**Параметры:**

* `employee_name`
* `settlement_status`
* `total_expense`
* `advance_given`

**Логика:**

* Рассчитать разницу: `difference_amount = abs(total_expense - advance_given)`.
* Создать и вернуть словарь:

```json
{
  "employee_name": "Орлов О.О.",
  "settlement_status": "Доплата сотруднику",
  "difference_amount": 5000.00
}
```

---

## Главный процесс должен:

1. Создать пустой список `final_expense_list`.
2. В цикле для каждого отчёта:

   * Вызвать `CalculateTotalExpense`;
   * Вызвать `DetermineSettlementStatus` (передать сумму расходов и аванс);
   * Вызвать `BuildExpenseRecord`;
   * Добавить возвращённый словарь в `final_expense_list`.
3. Сохранить `final_expense_list` в файл
   `expense_result.json`.
4. Проверить, что итоговый файл содержит **6 записей**.
5. Вывести итоговый список на экран.

---

## Архитектурные требования

* Все функции создаются через блок «Определение функции и возврат значения».
* Каждая функция реализует принцип «одна функция — одна ответственность».
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит вычислений.
* Используются списки и словари.
* Все результаты обязательно сохраняются в файл.
