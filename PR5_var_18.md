# Вариант 18

## Кейс: Автоматизация расчёта рейтинга торговых точек

В крупной розничной сети внедряется система автоматического формирования рейтинга торговых точек на основе ключевых показателей эффективности.
Руководство приняло решение исключить ручную агрегацию данных из разных систем и минимизировать риски субъективной оценки при распределении ресурсов между магазинами.

Ранее региональные менеджеры вручную собирали отчёты о выручке, трафике и удовлетворённости клиентов, рассчитывали интегральные баллы в таблицах. Процесс занимал значительное время в конце отчётного периода и сопровождался риском несогласованности методик расчёта.

В этом году принято решение автоматизировать процесс с использованием программного робота, разработанного в **Puzzle RPA**.

Отдел развития сети формирует список торговых точек с данными о показателях и передаёт его в виде JSON-файла.
Задача разработчика — спроектировать робота с использованием функций, который:

* корректно считывает данные из файла;
* структурированно обрабатывает список торговых точек;
* разделяет логику на самостоятельные модули;
* формирует новые списки результатов;
* сохраняет итоговые данные в отдельные JSON-файлы.

---

# Требования к исходным данным

1. Исходные данные необходимо **сгенерировать с помощью ИИ DeepSeek**.
2. Сформировать JSON-файл, содержащий **ровно 6 записей**.
3. Формат — список словарей.
4. Структура каждой записи:

```json
{
  "store_name": "ТЦ «Галерея»",
  "monthly_revenue": 2500000,
  "customer_satisfaction": 4.2,
  "staff_count": 15
}
```

5. Робот должен считать данные из созданного JSON-файла.

# 🔹 Задание 1 — Базовый уровень

## Этап 1: Проверка минимального порога выручки

### Ситуация

На первом этапе система должна проверить, достигает ли торговая точка минимального порога месячной выручки для участия в рейтинговании.
Порог установлен на уровне 1 000 000 рублей.

Отделу аналитики необходимо получить отдельный файл со статусом соответствия порогу по каждой торговой точке.

---

## Требуется:

1. Считать JSON-файл (6 торговых точек).
2. Организовать цикл обработки списка словарей.
3. Создать функцию с возвратом значения.

---

### Функция `CheckRevenueThreshold`

**Тип:** Определение функции и возврат значения
**Параметр:**

* `store_dict`

**Логика:**

* Получить значение `monthly_revenue` из словаря.
* Если `monthly_revenue` < 1000000 → вернуть `"Ниже порога"`.
* Если `monthly_revenue` ≥ 1000000 → вернуть `"Выше порога"`.

---

## Главный процесс должен:

1. Создать пустой список `revenue_check_list`.
2. В цикле:

   * Вызвать `CheckRevenueThreshold`;
   * Сформировать словарь:

```json
{
  "store_name": "ТЦ «Галерея»",
  "revenue_status": "Выше порога"
}
```

* Добавить словарь в `revenue_check_list`.

3. Сохранить `revenue_check_list` в файл
   `revenue_check.json`.
4. Вывести итоговый список на экран.

---

## Архитектурные требования

* Функция создаётся через блок «Определение функции и возврат значения».
* Функция выполняет только одну задачу — проверку порога выручки.
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит логики проверки.

---

# 🔹 Задание 2 — Повышенный уровень

## Этап 2: Расчёт интегрального рейтинга и формирование категории

### Ситуация

После проверки порога выручки руководство решило автоматизировать расчёт комплексного рейтинга торговых точек.

Теперь требуется не только подтвердить соответствие порогу, но и рассчитать интегральный балл на основе трёх факторов (выручка, удовлетворённость клиентов, численность персонала), а также присвоить точке категорию эффективности.

Отделу развития сети нужен структурированный список с расчётными данными для принятия решений о развитии и инвестициях.

---

## Требуется:

1. Использовать исходный JSON-файл из 6 торговых точек.
2. Организовать цикл обработки списка.
3. Создать три функции с возвратом значения.

---

### 1️⃣ Функция `CalculateStoreRating`

**Параметр:**

* `store_dict`

**Логика:**

* Получить `monthly_revenue`, `customer_satisfaction`, `staff_count`.
* Рассчитать рейтинг по формуле:
  `(monthly_revenue / 500000) + (customer_satisfaction * 2) + (staff_count * 0.3)`
* Вернуть число (интегральный рейтинг).

---

### 2️⃣ Функция `DetermineStoreCategory`

**Параметр:**

* `rating_value`

**Логика:**

* Если `rating_value` ≥ 15 → вернуть `"Лидер"`
* Если `rating_value` ≥ 10 → вернуть `"Стабильный"`
* Иначе → вернуть `"Требует внимания"`

---

### 3️⃣ Функция `BuildStoreRecord`

**Параметры:**

* `store_name`
* `category_label`
* `rating_score`

**Логика:**

* Создать и вернуть словарь:

```json
{
  "store_name": "ТЦ «Галерея»",
  "store_category": "Лидер",
  "rating_score": 16.4
}
```

---

## Главный процесс должен:

1. Создать пустой список `final_store_list`.
2. В цикле для каждой торговой точки:

   * Вызвать `CalculateStoreRating`;
   * Передать результат в `DetermineStoreCategory`;
   * Вызвать `BuildStoreRecord`;
   * Добавить возвращённый словарь в `final_store_list`.
3. Сохранить `final_store_list` в файл
   `store_rating_result.json`.
4. Проверить, что итоговый файл содержит **6 записей**.
5. Вывести итоговый список на экран.

---

## Архитектурные требования

* Все функции создаются через блок «Определение функции и возврат значения».
* Каждая функция реализует принцип «одна функция — одна ответственность».
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит вычислений.
* Используются списки и словари.
* Все результаты обязательно сохраняются в файл.
