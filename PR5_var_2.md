# Вариант 2

## Кейс: Автоматизация анализа заявок на отпуск

В крупной компании внедряется система автоматической обработки заявок на ежегодный оплачиваемый отпуск.
Руководство приняло решение оптимизировать работу HR-отдела, исключив ручную проверку остатков дней отпуска у сотрудников.

Ранее специалисты вручную сверяли заявки с таблицами учёта рабочего времени, что занимало много времени и приводило к ошибкам при расчёте остатков.

В этом году принято решение автоматизировать процесс с использованием программного робота, разработанного в **Puzzle RPA**.

HR-отдел формирует список заявок и передаёт его в виде JSON-файла.
Задача разработчика — спроектировать робота с использованием функций, который:

* корректно считывает данные из файла;
* структурированно обрабатывает список заявок;
* разделяет логику на самостоятельные модули;
* формирует новые списки результатов;
* сохраняет итоговые данные в отдельные JSON-файлы.

---

# Требования к исходным данным

1. Исходные данные необходимо **сгенерировать с помощью ИИ DeepSeek**.
2. Сформировать JSON-файл, содержащий **ровно 6 записей**.
3. Формат — список словарей.
4. Структура каждой записи:

```json
{
  "name": "Сидоров С.С.",
  "leave_balance": 14,
  "days_requested": 10
}
```

5. Робот должен считать данные из созданного JSON-файла.

---

# 🔹 Задание 1 — Базовый уровень

## Этап 1: Проверка достаточности дней отпуска

### Ситуация

На первом этапе система должна проверить, хватает ли у сотрудника накопленных дней отпуска для удовлетворения заявки.

HR-отделу необходимо получить отдельный файл со статусом доступности дней по каждой заявке.

---

## Требуется:

1. Считать JSON-файл (6 заявок).
2. Организовать цикл обработки списка словарей.
3. Создать функцию с возвратом значения.

---

### Функция `CheckLeaveBalance`

**Тип:** Определение функции и возврат значения
**Параметр:**

* `request_dict`

**Логика:**

* Получить значения `leave_balance` и `days_requested` из словаря.
* Если `days_requested` > `leave_balance` → вернуть `"Недостаточно средств"`.
* Если `days_requested` ≤ `leave_balance` → вернуть `"Достаточно средств"`.

---

## Главный процесс должен:

1. Создать пустой список `balance_check_list`.
2. В цикле:

   * Вызвать `CheckLeaveBalance`;
   * Сформировать словарь:

```json
{
  "name": "Сидоров С.С.",
  "balance_status": "Достаточно средств"
}
```

* Добавить словарь в `balance_check_list`.

3. Сохранить `balance_check_list` в файл
   `balance_check.json`.
4. Вывести итоговый список на экран.

---

## Архитектурные требования

* Функция создаётся через блок «Определение функции и возврат значения».
* Функция выполняет только одну задачу — проверку баланса дней.
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит логики проверки.

---

# 🔹 Задание 2 — Повышенный уровень

## Этап 2: Формирование итогового решения по заявке

### Ситуация

После проверки баланса руководство решило усложнить процесс согласования.

Теперь требуется не только проверить дни, но и рассчитать остаток после отпуска, а также присвоить финальный статус заявки для передачи в бухгалтерию.

HR-отделу нужен структурированный список заявок с расчётными данными для автоматического формирования приказов.

---

## Требуется:

1. Использовать исходный JSON-файл из 6 заявок.
2. Организовать цикл обработки списка.
3. Создать три функции с возвратом значения.

---

### 1️⃣ Функция `CalculateRemainingDays`

**Параметр:**

* `request_dict`

**Логика:**

* Получить `leave_balance` и `days_requested`.
* Рассчитать остаток:
  `leave_balance - days_requested`
* Вернуть число (может быть отрицательным).

---

### 2️⃣ Функция `DetermineApprovalStatus`

**Параметр:**

* `remaining_days`

**Логика:**

* Если `remaining_days` ≥ 0 → вернуть `"Согласовано"`
* Иначе → вернуть `"Отклонено"`

---

### 3️⃣ Функция `BuildLeaveRecord`

**Параметры:**

* `employee_name`
* `status_text`
* `remaining_days`

**Логика:**

* Создать и вернуть словарь:

```json
{
  "name": "Сидоров С.С.",
  "final_status": "Согласовано",
  "remaining_balance": 4
}
```

---

## Главный процесс должен:

1. Создать пустой список `final_approval_list`.
2. В цикле для каждой заявки:

   * Вызвать `CalculateRemainingDays`;
   * Передать результат в `DetermineApprovalStatus`;
   * Вызвать `BuildLeaveRecord`;
   * Добавить возвращённый словарь в `final_approval_list`.
3. Сохранить `final_approval_list` в файл
   `approval_result.json`.
4. Проверить, что итоговый файл содержит **6 записей**.
5. Вывести итоговый список на экран.

---

## Архитектурные требования

* Все функции создаются через блок «Определение функции и возврат значения».
* Каждая функция реализует принцип «одна функция — одна ответственность».
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит вычислений.
* Используются списки и словари.
* Все результаты обязательно сохраняются в файл.
