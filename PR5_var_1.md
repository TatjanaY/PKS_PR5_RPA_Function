# Вариант 1

## Кейс: Автоматизация отбора сотрудников в кадровый резерв

Крупная производственная компания внедряет программу развития кадрового резерва.
Руководство приняло решение ежегодно формировать список сотрудников, потенциально готовых к выдвижению на руководящие должности.

Ранее отбор проводился вручную: HR-специалисты анализировали стаж работы, годовые рейтинги и количество успешных проектов. Процесс занимал несколько дней и сопровождался риском субъективной оценки.

В этом году принято решение автоматизировать процесс с использованием программного робота, разработанного в **Puzzle RPA**.

HR-отдел формирует список сотрудников и передаёт его в виде JSON-файла.
Задача разработчика — спроектировать робота с использованием функций, который:

* корректно считывает данные из файла;
* структурированно обрабатывает список сотрудников;
* разделяет логику на самостоятельные модули;
* формирует новые списки результатов;
* сохраняет итоговые данные в отдельные JSON-файлы.

---

# Требования к исходным данным

1. Исходные данные необходимо **сгенерировать с помощью ИИ DeepSeek**.
2. Сформировать JSON-файл, содержащий **ровно 6 записей**.
3. Формат — список словарей.
4. Структура каждой записи:

```json
{
  "name": "Петров П.П.",
  "years_of_service": 5,
  "annual_rating": 4.5,
  "completed_projects": 10
}
```

5. Робот должен считать данные из созданного JSON-файла.

---

# 🔹 Задание 1 — Базовый уровень

## Этап 1: Проверка минимального стажа

### Ситуация

На первом этапе руководство установило базовое требование:
в кадровый резерв могут быть рассмотрены только сотрудники со стажем работы в компании не менее 3 лет.

HR-отделу необходимо получить отдельный файл со статусом допуска по каждому сотруднику.

---

## Требуется:

1. Считать JSON-файл (6 сотрудников).
2. Организовать цикл обработки списка словарей.
3. Создать функцию с возвратом значения.

---

### Функция `CheckServiceYears`

**Тип:** Определение функции и возврат значения
**Параметр:**

* `employee_dict`

**Логика:**

* Получить значение `years_of_service` из словаря.
* Если стаж < 3 → вернуть `"Не соответствует"`.
* Если стаж ≥ 3 → вернуть `"Соответствует"`.

---

## Главный процесс должен:

1. Создать пустой список `tenure_result_list`.
2. В цикле:

   * Вызвать `CheckServiceYears`;
   * Сформировать словарь:

```json
{
  "name": "Петров П.П.",
  "tenure_status": "Соответствует"
}
```

* Добавить словарь в `tenure_result_list`.

3. Сохранить `tenure_result_list` в файл
   `tenure_result.json`.
4. Вывести итоговый список на экран.

---

## Архитектурные требования

* Функция создаётся через блок «Определение функции и возврат значения».
* Функция выполняет только одну задачу — проверку стажа.
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит логики проверки.

---

# 🔹 Задание 2 — Повышенный уровень

## Этап 2: Формирование списка кандидатов в резерв

### Ситуация

После проверки стажа руководство компании решило усложнить процесс отбора.

Теперь требуется учитывать эффективность работы сотрудников.
Было принято решение:

* оценивать совокупный балл на основе рейтинга и проектов;
* формировать рекомендацию на основании расчётного scores;
* создать отдельный итоговый файл с результатами второго этапа отбора.

HR-отделу нужен структурированный список сотрудников со статусом включения в резерв для передачи высшему руководству.

---

## Требуется:

1. Использовать исходный JSON-файл из 6 сотрудников.
2. Организовать цикл обработки списка.
3. Создать три функции с возвратом значения.

---

### 1️⃣ Функция `CalculateReserveScore`

**Параметр:**

* `employee_dict`

**Логика:**

* Получить `annual_rating` и `completed_projects`.
* Если `completed_projects` = 0 → вернуть 0.
* Рассчитать коэффициент:
  `(annual_rating * completed_projects) / 10`
* Вернуть число.

---

### 2️⃣ Функция `DetermineReserveStatus`

**Параметр:**

* `score_value`

**Логика:**

* Если коэффициент ≥ 5.0 → вернуть `"Включён в резерв"`
* Иначе → вернуть `"Не включён"`

---

### 3️⃣ Функция `BuildReserveRecord`

**Параметры:**

* `employee_name`
* `status_text`

**Логика:**

* Создать и вернуть словарь:

```json
{
  "name": "Петров П.П.",
  "reserve_status": "Включён в резерв"
}
```

---

## Главный процесс должен:

1. Создать пустой список `final_reserve_list`.
2. В цикле для каждого сотрудника:

   * Вызвать `CalculateReserveScore`;
   * Передать результат в `DetermineReserveStatus`;
   * Вызвать `BuildReserveRecord`;
   * Добавить возвращённый словарь в `final_reserve_list`.
3. Сохранить `final_reserve_list` в файл
   `reserve_result.json`.
4. Проверить, что итоговый файл содержит **6 записей**.
5. Вывести итоговый список на экран.

---

## Архитектурные требования

* Все функции создаются через блок «Определение функции и возврат значения».
* Каждая функция реализует принцип «одна функция — одна ответственность».
* Запрещено использовать глобальные переменные.
* Главный процесс не содержит вычислений.
* Используются списки и словари.
* Все результаты обязательно сохраняются в файл.
